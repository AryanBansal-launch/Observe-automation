Dashboard check, Fix flow, hostname lookup, and Slack for Observe.

About this app & how to use it

What it does

This app extracts error reports from Observe log datasets, lets you analyze and fix individual errors with AI, look up hostnames to find deployments, and send formatted reports to Slack.

Features

  • Dashboard Check – Run error extraction for one service, all services, or all regions. Results appear in a table with links to Observe Log Explorer.
  • Fix – Click Fix next to any error to run the Cursor agent on that single error. Output goes to output/agent_analysis.md. Requires Cursor CLI and AGENT_WORKSPACE in .env.
  • Lookup by Hostname – Enter a hostname or URL to get orgUid, projectUid, environmentUid, region, and deployment info from Nginx + Mgmt data.
  • Send me a Slack – Format the report with Gemini and POST to your webhook. Set SLACK_WEBHOOK_URL and GEMINI_API_KEY in .env.

Quick start

1. Enter Customer ID and API Key below. 2. Choose a run mode and click Run dashboard check. 3. Use Copy, Download, Fix, or Send me a Slack on the report.

Credentials How do I get API key & Customer ID?

Customer ID (OBSERVE_CUSTOMER_ID)

1. Open your Observe workspace in the browser (e.g. your Observe URL).
2. The Customer ID is the first part of the URL before .observeinc.com — the numbers (and sometimes cluster).
Example: from https://143110822295.eu-1.observeinc.com/workspace/... → Customer ID is 143110822295 (the numeric part before the cluster).

API token (OBSERVE_API_KEY)

1. In Observe, go to Settings → My API tokens (or open https://<your-customer-id>.<cluster>.observeinc.com/settings/my-api-tokens).
2. Create a new API token and copy it — it may not be shown again.
3. Ensure the token has dataset:view (or equivalent) on the datasets you query.

Dashboard Check How to use it?

Single service

Run the dashboard check for one service only. Select the service from the dropdown below. Uses the region from the optional Environment vars (default aws-na). Best for quick checks on a specific service.

All services

Run the check for all services defined in the app config, in the current region. Use this to see errors across every service in one run.

All regions

Run the check for all services across all regions (e.g. aws-na, aws-eu, azure-na, azure-eu, gcp-na, gcp-eu, aws-au). Produces a combined report grouped by region. Can take longer.

Auto (all services × all regions)

Same as All regions: runs the dashboard check for every service in every region. Use when you want a full sweep. May take several minutes.

Report

What is this?

After you run a dashboard check, Send me a Slack formats the report and posts it to the webhook URL configured in SLACK_WEBHOOK_URL (env).

How to use

1. Run a dashboard check first so there is a report.
2. (Optional) Enter a Channel ID — e.g. C1234567890 or D086ZCDT6B0 to override the webhook's default channel.
3. Click Send me a Slack. The app uses Gemini to format the report, then POSTs it to your webhook (or copies to clipboard if no webhook is configured).

Lookup by Hostname How to use it?

Step 1: Enter credentials

In the Credentials section above, enter your Customer ID and API Key. Optionally set OBSERVE_CLUSTER (e.g. eu-1) if your Observe instance uses a regional cluster.

Step 2: Load data

Click Load data. The app fetches Nginx and deployment data from Observe. Wait until you see "Data loaded. You can run lookups now."

Step 3: Look up a hostname

Enter a hostname or full URL (e.g. www.abc.com or https://www.abc.com/path) and click Look up. You get orgUid, projectUid, environmentUid, region, and the latest deployment UID and timestamp.

Tips

If you skip Load data and click Look up directly, the first lookup will fetch data and may take longer. Subsequent lookups use cached data for a few minutes.
