Environment How do I get API key & Customer ID?

Customer ID (OBSERVE_CUSTOMER_ID)

1. Open your Observe workspace in the browser (e.g. your Observe URL).
2. The Customer ID is the first part of the URL before .observeinc.com — the numbers (and sometimes cluster).
Example: from https://143110822295.eu-1.observeinc.com/workspace/... → Customer ID is 143110822295 (the numeric part before the cluster).

API token (OBSERVE_API_KEY)

1. In Observe, go to Settings → My API tokens (or open https://<your-customer-id>.<cluster>.observeinc.com/settings/my-api-tokens).
2. Create a new API token and copy it — it may not be shown again.
3. Ensure the token has dataset:view (or equivalent) on the datasets you query.

Look up by hostname How to use it?

Step 1: Enter credentials

In the Environment section above, enter your Customer ID and API Key. Optionally set OBSERVE_CLUSTER (e.g. eu-1) if your Observe instance uses a regional cluster; otherwise the server uses the value from its env.

Step 2: Load data

Click Load data. The app fetches Nginx and deployment data from Observe (time range and cluster come from server env). Wait until you see "Data loaded. You can run lookups now."

Step 3: Look up a hostname

Enter a hostname or full URL (e.g. www.abc.com or https://www.abc.com/path) in the box and click Look up. You get orgUid, projectUid, environmentUid, region, and the latest deployment UID and timestamp for that hostname.

Tips

If you skip Load data and click Look up directly, the first lookup will fetch data and may take longer. Subsequent lookups use cached data for a few minutes. You can run Load data again to refresh the cache.

Run mode How to use it?

Single service

Run the dashboard check for one service only. Select the service from the dropdown below. Uses the region from the optional Environment vars (default aws-na). Best for quick checks on a specific service.

All services

Run the check for all services defined in the app config, in the current region. Use this to see errors across every service in one run.

All regions

Run the check for all services across all regions (e.g. aws-na, aws-eu, azure-na, azure-eu, gcp-na, gcp-eu, aws-au). Produces a combined report grouped by region. Can take longer.

Auto (all services × all regions)

Same as All regions: runs the dashboard check for every service in every region. Use when you want a full sweep. May take several minutes.

Tip

After choosing a mode, click Run dashboard check below. The report appears in the Report section; you can then Copy, Download, or Send me a Slack.

Report & actions

Report

What is this?

After you run a dashboard check, Send me a Slack formats the report and can post it to a webhook URL (e.g. Slack incoming webhook, Microsoft Teams, or any HTTP endpoint that accepts JSON).

How to use

1. Run a dashboard check first so there is a report.
2. (Optional) Enter a Webhook URL — e.g. a Slack Incoming Webhook URL, or leave blank to only format and copy the message.
3. Click Send me a Slack. The app uses Gemini to format the report, then either POSTs it to your webhook or copies the formatted message to the clipboard.

Without a webhook URL

If you leave the Webhook URL empty, the formatted message is copied to your clipboard and shown in the status. You can paste it into Slack or anywhere else manually.
